# Disassembler Design

Status: Draft complete (Phase 3.2.1)
Owner: Active
Last updated: 2026-04-06

## Goal

Define a practical, incremental design for disassembling x86_64 code generated by this assembler.

The first implementation target is correctness for assembler-emitted binaries, then progressive expansion toward broader external binary compatibility.

## CLI Design

## Command

Add a disassembly mode to the existing CLI:

- `--disassemble <input-binary>`
- Short alias: `-D <input-binary>` (chosen to avoid conflict with existing `-d` dump-symbols flag)

Examples:

```bash
./bin/x86_64-asm --disassemble a.out
./bin/x86_64-asm -D a.out
```

## Behavior

- Input is an existing ELF64 executable/object.
- The tool reads `.text` and decodes instructions sequentially.
- Output is printed to stdout.
- Non-fatal decode gaps should render as data bytes (`db 0x..`) and continue.
- Fatal file/format errors return non-zero with standardized diagnostics.

## Input/Output Scope

## Input Scope (Phase 1 of disassembler)

- ELF64 little-endian.
- Primarily binaries produced by this assembler.
- Requires valid section table and `.text` availability.

## Output Scope

Human-readable assembly-like listing with address and bytes:

```text
0000000000400000: 48 c7 c0 3c 00 00 00    mov rax, 0x3c
0000000000400007: 48 31 ff                xor rdi, rdi
000000000040000a: 0f 05                   syscall
```

Formatting rules:

- Address: 16-hex lowercase with `0x` omitted in left column.
- Bytes: lowercase hex, space-separated, fixed print width region.
- Mnemonic: canonical lowercase.
- Operands: Intel-style (consistent with assembler input style).

## Architecture

## Pipeline

1. Parse ELF header.
2. Locate `.text` section.
3. Build decode cursor at section start.
4. Decode one instruction at a time:
   - Prefixes (legacy + REX)
   - Primary opcode bytes (including escape maps `0F`, `0F 38`, `0F 3A` as needed)
   - ModRM/SIB/disp/immediate fields
5. Render instruction text.
6. Advance cursor by decoded length.

## Error Handling Strategy

- File/ELF structure errors: fatal, standardized diagnostic.
- Unknown opcode at cursor: emit `db 0x??` for one byte and continue.
- Truncated instruction tail at end of section: emit remaining bytes as `db` and stop.

## Data Structures

## Decoder Context

```c
typedef struct {
    const uint8_t *text;
    size_t text_size;
    uint64_t text_vaddr;
    size_t offset;
} dis_ctx_t;
```

## Decoded Instruction

```c
typedef struct {
    uint64_t address;
    uint8_t bytes[15];
    int byte_count;
    char mnemonic[32];
    char operands[128];
    bool valid;
} decoded_inst_t;
```

## Opcode Lookup Entry (initial)

```c
typedef struct {
    uint16_t opcode_key;   /* e.g. primary + map marker */
    const char *mnemonic;
    uint8_t operand_pattern;
    uint8_t flags;
} opcode_entry_t;
```

Notes:
- Start with table-driven decode for common fixed forms.
- Use focused handlers for families requiring ModRM interpretation.

## Instruction Family Rollout

## Slice A (minimum useful)

- `mov` variants used by assembler baseline:
  - `mov r64, imm32/imm64`
  - `mov r/m64, r64`
  - `mov r64, r/m64`
- arithmetic/logical core:
  - `add`, `sub`, `cmp`, `xor`, `and`, `or` (reg/mem + imm where already emitted)
- control/sys:
  - `jmp rel32`, `call rel32`, `ret`, `syscall`
- unary basics:
  - `inc`, `dec`, `not`, `neg`

Exit criterion:
- Correctly disassemble simple runtime samples (`hello`, `exit`, arithmetic/branch basics).

## Slice B (flow and addressing depth)

- Jcc near forms (`0F 8x`).
- `lea`.
- push/pop register and common memory forms.
- robust ModRM/SIB/disp decoding coverage.

Exit criterion:
- Disassembler round-trips control-flow examples and SIB-heavy encoder fixtures.

## Slice C (extended integer families)

- `setcc`, `cmovcc`.
- bit families used in tests:
  - `bt/bts/btr/btc`
  - `bsf/bsr`
  - `shld/shrd`
- `imul`/`mul`/`div`/`idiv` forms currently emitted.

Exit criterion:
- Decoder passes integer-family comparison fixtures against expected mnemonics and operand forms.

## Slice D (SSE baseline)

- SSE move/arithmetic/logical/packed families currently encoded in project.
- Prefix-map aware decode (`66`, `F2`, `F3`, `0F`, `0F38`, `0F3A` where needed).

Exit criterion:
- SSE integration fixtures disassemble with correct mnemonic families and operand routing.

## Validation Strategy

## Unit Tests

- Decoder cursor and field extraction tests.
- ModRM/SIB decode tests for representative addressing modes.
- Opcode map dispatch tests.

## Integration Tests

- Assemble -> disassemble on project examples.
- Compare normalized disassembly against expected canonical snippets.
- Cross-check selected binaries against `objdump -d` mnemonic/operand equivalence (not strict text formatting equality).

## Metrics

- Coverage target for decoder core: >= 85% branch coverage in implemented slices.
- Unknown opcode fallback rate should be 0 for assembler-emitted binaries in covered slices.

## Risks and Decisions

## Risks

- x86_64 decode complexity explosion if all encodings are tackled at once.
- Prefix/opcode interaction corner cases in SSE and extended maps.
- Formatting differences vs `objdump` can obscure semantic correctness if compared textually.

## Decisions

- Incremental slice strategy over full ISA decode.
- Semantic comparison in tests preferred over strict full-line string equality.
- Unknown opcode is non-fatal in disassemble mode unless file-level structure is invalid.

## Out of Scope (for 3.2.1)

- Full external binary compatibility beyond covered slices.
- Relocation/symbol-aware annotated disassembly output.
- Control-flow graph generation.
