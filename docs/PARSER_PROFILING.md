# Parser Profiling Report

Status: Completed (Phase 3.3.1/3.3.2/3.3.3)
Owner: Active
Last updated: 2026-04-07

## Scope

This report documents parser performance profiling on a synthetic large-input workload and validates symbol lookup performance between linear scan and hash-table lookup.

## Method

Profiling command:

```bash
make profile-parser
```

Workload generated by `tools/profile_parser.c`:
- 3000 instruction blocks
- About 15006 source lines
- 5 parse runs

Instrumentation added in `src/x86_64_parser.c`:
- `parse_source_internal` wall time
- `parse_instruction` aggregate time and call count
- `next_token` aggregate time and call count
- instruction-buffer realloc event count
- peak instruction-buffer capacity

## Baseline Results

Latest run output:

- Average parse time: 158.540 ms
- `parse_source_internal`: 790.483 ms across 5 runs
- `parse_instruction`: 780.944 ms across 75025 calls
- `next_token`: 24.693 ms across 525100 calls
- Instruction buffer realloc events: 0
- Peak instruction buffer capacity: 16913

Interpretation:
- Parsing cost is dominated by per-instruction parsing logic.
- Tokenization is measurable but not the dominant hotspot.
- Buffer growth churn was eliminated in this workload (`realloc_events = 0`).

## Symbol Lookup Benchmark

The same profiling target includes a lookup microbenchmark with 3500 symbols and 400000 lookups:

- Linear lookup: 1644.280 ms
- Hash lookup: 25.810 ms
- Speedup: 63.71x

Checksum parity confirms identical lookup resolution behavior in the benchmark path.

## Implemented Optimizations

### 1) Parser profiling instrumentation (3.3.1)
- Added parser profiling API:
  - `parser_profile_enable(...)`
  - `parser_profile_reset()`
  - `parser_profile_get(...)`
- Added internal timing/counters for parser and tokenizer hot paths.

### 2) Symbol lookup optimization validation (3.3.2)
- Existing hash-table symbol lookup path (`symbol_hash_lookup`) benchmarked against linear scan.
- Measured speedup exceeds target (10x+) for large symbol counts.

### 3) Token and line-buffer optimization (3.3.3)
- Added instruction-buffer pre-sizing in `parse_source_internal(...)` based on source line count estimate.
- Confirmed allocation churn reduction via profiling counters (no reallocations in benchmark workload).

## Regression Validation

After the profiling/optimization changes:

- `make test-parser` → 80/80 pass
- `make test-encoder` → 138/138 pass
- `make test-disassembler` → 16/16 pass
- `make test-integration` → 57/57 pass

## Conclusion

Phase 3 parser-performance milestones are satisfied with reproducible measurements, validated speedup for symbol lookup, reduced allocation churn, and full regression pass.
